# DESCRIPTION

Complete API requirements for 128 REST endpoints across 13 modules with RBAC permission matrix and 142 UI screen specifications. Maps user actions to required data (GET) and data input (POST/PATCH) for each screen. Use when creating controllers, DTOs, guards, or understanding endpoint-to-screen mappings.

**Scope:**
- 13 backend modules (resource-based architecture)
- 128 REST endpoints  
- 142 UI screens mapped to API requirements
- Complete RBAC permission matrix (User, Player, Captain, Vice Captain, Admin)

**Related Documentation:**
- `prd.mdc` - Product requirements and feature specifications
- `game-rules.mdc` - Business rules and validation logic

**UI Pattern (iconRef vs URLs):**
- **iconRef**: Standard UI icons bundled in frontend (e.g., `"iconRef": "warning_triangle"`)
- **URLs**: Dynamic content from backend CDN (e.g., `"squadLogoUrl": "https://cdn.squadz.com/..."`)

# ROLE DEFINITIONS

| Role | Description | How Assigned |
|------|-------------|--------------|
| **User** | Incomplete account setup; restricted access | Default after registration |
| **Player** | Completed account setup; can participate | Auto-assigned after setup completion |
| **Captain** | Squad owner; full team management | Created squad OR assigned by Captain |
| **Vice Captain** | Assists Captain with match operations | Assigned by Captain |
| **Admin** | Platform administrator; system management | Manual assignment |

**Permission Inheritance:**
- Player inherits all User permissions
- Captain inherits all Player permissions
- Vice Captain inherits all Player permissions
- Admin has override access to all endpoints

# PERMISSION VALUES

| Value | Meaning | Implementation |
|-------|---------|----------------|
| **YES** | Full access | No additional guards beyond authentication |
| **NO** | No access (403) | Block with role guard |
| **OWN** | Own resource only | Validate `userId` in token matches `:id` in route |
| **SQUAD** | Own squad only | Validate user is Captain of `squadId` |
| **CONDITIONAL** | Business rules apply | See "Conditions" column in permission matrix |
